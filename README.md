# Paint Locker（塗料ストック管理アプリ）

模型用塗料（市販品・調色の自作色）のコレクションを管理するWebアプリです。
ビルド工程なし。`index.html` を開くだけで動きます。

## 機能
- メーカー → シリーズ の階層で整理・グループ表示（折りたたみ対応）
- よく使うメーカーのピン留め（画面上部に固定）
- 本数の増減、残量（%）のメモ
- ビュー切替：すべて / 所持（本数1以上）/ 調色
- 調色塗料の登録と配合（材料・比率・メモ）の記録
- スマホカメラでバーコード読取（初回登録方式）
- データのJSONバックアップ書き出し

## 起動方法

### そのまま開く
`index.html` をブラウザで開くだけです。
※ ただし `file://` で開くとカメラが起動しないことがあります。バーコード読取を使うなら下記のローカルサーバ経由を推奨します。

### ローカルサーバで開く（バーコード読取を使う場合）
カメラはHTTPS、または `localhost` でのみ動きます。

```bash
# どちらでも可
python3 -m http.server 8000
# または
npx serve .
```

ブラウザで `http://localhost:8000` を開きます。
スマホで使う場合は GitHub Pages 等のHTTPS環境に置くのが簡単です。

### GitHub Pages に置く
リポジトリにこの一式を push し、Settings → Pages で公開すると、スマホからHTTPSで利用できます。

## データの保存先
- 既定は **localStorage**（その端末・そのブラウザの中）です。
- **複数端末の自動同期はしません。** PC とスマホは別々のデータになります。
- 同期したい場合は `storage.js` の `load` / `save` をサーバAPIに差し替えてください（詳細は SPEC.md 第6章）。
- 当面は「バックアップ書き出し(JSON)」で手動バックアップが取れます。

## ファイル構成
```
index.html   画面の骨組み
styles.css   スタイル
storage.js   データ永続化層（差し替え前提）
app.js       画面描画とロジック
SPEC.md      仕様書（変更の基準）
README.md    このファイル
```

依存：html5-qrcode（CDN読み込み）のみ。

## Claude Code での進め方

このプロジェクトは Claude Code で仕様とコードを管理する前提で作っています。

### 最初の一歩
1. このフォルダを Git リポジトリにします。
   ```bash
   git init
   git add .
   git commit -m "初期コミット: Paint Locker"
   ```
2. リポジトリ直下で `claude` を起動します。
3. まず仕様を読ませます。
   ```
   SPEC.md を読んで、現状の実装と仕様の差分があれば教えてください
   ```

### 変更の進め方（おすすめの型）
- 機能追加は「まず SPEC.md を直す → それに合わせてコードを直す」の順にすると、仕様とコードがズレません。
  ```
  SPEC.md に「残量を満タン/半分/わずか の3段階プリセットボタンで設定できる」を追記し、
  app.js の編集フォームに反映してください
  ```
- 1コミット1目的を意識すると、後から差分を追いやすくなります。

### よく使う依頼の例
- 「複数端末で同期したいので、storage.js を Supabase 版に差し替える計画を SPEC.md に書いてください」
- 「JSONインポート機能（バックアップから復元）を追加してください」
- 「調色の材料を、既存塗料のIDで参照できるようにしてください」

### 注意点
- `storage.js` の `load` / `save` の「形（async関数）」は変えないでください。ここを保てば保存方式を自由に差し替えられます。
- データモデルを変える時は SPEC.md 第3章を必ず更新してください。
